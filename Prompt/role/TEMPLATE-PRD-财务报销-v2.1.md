## 写作与排版规范

### 排版

| 项 | 要求 |
| --- | --- |
| 标题 | 使用 `#` `##` `###` 层级，与本文档一致 |
| 表格 | 表头与内容左对齐；关键表头可加粗 |
| 示意图 | 界面原型用 ` ```text ` 代码块包裹 ASCII 图 |
| 列表 | 使用 `-` 或 `•`，同一文档内统一 |
| 边界 | 一律写在 **§3.3**，不在各功能点小节重复 |

### 文字风格

- 专业、简练、可执行；避免同义反复与套话。
- **不清楚处**：标「待确认」并列出待澄清问题，不擅自补充业务事实。

### 交付物要求

- 交付完整 PRD 正文，**章节齐全**；不适用的章节写「本期不涉及」或「无」。
- 对外评审前自检：元数据、变更清单、§3.3 边界、验收标准可逐条对照。
