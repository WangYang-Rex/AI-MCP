# 灯塔工厂概念股实时行情脚本

> 基于东方财富API的股票行情查询工具，已按 `eastmoney.md` 开发建议进行优化

## ✨ 优化功能

### 1. 请求优化
- ✅ **完整的请求头**：添加 User-Agent、Referer、Accept 等头部，模拟真实浏览器请求
- ✅ **请求超时控制**：10秒超时保护
- ✅ **自动重试机制**：失败后自动重试3次，每次间隔1秒
- ✅ **Gzip解压支持**：自动识别并解压 gzip/deflate 响应

### 2. 数据处理
- ✅ **数据验证**：严格的数据格式和结构验证
- ✅ **智能缓存**：1分钟缓存机制，减少API调用
- ✅ **多字段支持**：扩展显示振幅、市盈率、最高最低价等字段
- ✅ **数据过滤**：自动过滤停牌和无效数据

### 3. 输出优化
- ✅ **CSV输出**：标准CSV格式，方便Excel导入
- ✅ **JSON输出**：结构化JSON格式，方便程序调用
- ✅ **美化展示**：彩色表格输出，可读性更强
- ✅ **统计信息**：自动统计涨跌数量、平均涨跌幅、成交量等

### 4. 错误处理
- ✅ **分类错误信息**：区分错误类型和错误消息
- ✅ **DEBUG模式**：设置 `DEBUG=1` 显示详细错误栈
- ✅ **优雅退出**：错误时返回非零状态码

## 📦 运行要求

- **Node.js**: >= 12.x
- **依赖**: 零第三方依赖，仅使用原生模块

## 🚀 使用方法

### 基本用法

```bash
# 运行脚本
node lighthouse.js

# DEBUG模式（显示详细错误信息）
DEBUG=1 node lighthouse.js
```

### 配置说明

脚本顶部的 `CONFIG` 对象可自定义配置：

```javascript
const CONFIG = {
  // 请求配置
  REQUEST_TIMEOUT: 10000,        // 请求超时时间(ms)
  MAX_RETRIES: 3,                // 最大重试次数
  RETRY_DELAY: 1000,             // 重试间隔(ms)
  
  // 缓存配置
  CACHE_ENABLED: true,           // 是否启用缓存
  CACHE_TTL: 60 * 1000,          // 缓存有效期(ms) - 1分钟
  CACHE_FILE: 'lighthouse.cache.json',
  
  // 输出配置
  OUTPUT_CSV: 'lighthouse.csv',
  OUTPUT_JSON: 'lighthouse.json',
};
```

### 自定义股票列表

修改 `stocks` 数组添加或删除股票：

```javascript
const stocks = [
  { code: '600588', market: 1 },  // 上交所/科创板：market = 1
  { code: '002230', market: 0 },  // 深交所：market = 0
  // 添加更多股票...
];
```

## ⚠️ 注意事项

### 1. 法律合规
- ⚠️ 非官方API接口，仅供个人学习研究使用
- ❌ 禁止用于商业用途
- ✅ 建议添加请求间隔，避免给服务器造成压力

### 2. 数据准确性
- 📊 数据来源：东方财富网
- ⏰ 可能存在延迟，非Level-2实时数据
- 🔄 建议与官方数据交叉验证

### 3. 接口稳定性
- 🔧 接口可能随时变化或失效
- 💾 建议启用缓存机制
- 🔄 失败时会自动重试3次

### 4. 性能建议
- 📈 首次运行：~145ms（网络请求）
- ⚡ 缓存命中：~16ms（本地读取）
- 💡 缓存有效期默认1分钟，可根据需求调整

## 🛠️ 故障排查

### 问题1：返回数据为空
**原因**：股票代码或市场代码错误
**解决**：检查 `stocks` 数组中的代码和市场代码是否正确

### 问题2：请求超时
**原因**：网络问题或接口不可用
**解决**：
- 检查网络连接
- 增加 `REQUEST_TIMEOUT` 时间
- 查看是否被限流

### 问题3：接口返回异常
**原因**：接口结构变化或参数错误
**解决**：
- 运行 `DEBUG=1 node lighthouse.js` 查看详细错误
- 参考 `eastmoney.md` 文档检查最新接口

### 问题4：缓存数据不更新
**解决**：删除 `lighthouse.cache.json` 文件重新运行
